## Compression-Safe Summary
- Documentation-first workflow is accepted.
- Static GitHub Pages deployment is the provisional target.
- Data should be processed locally in the browser by default.
- Chart and spreadsheet behavior should use mature libraries after evaluation.
- The active GUI baseline avoids user-facing `replicate`, `series`, `solo/isolate`, and exclude/restore wording unless the user later confirms those concepts.
- The active GUI baseline treats specimen/reagent grouping as alternate views over the same selected curve IDs, not as separate selection states.
- The active GUI baseline uses reagent-first analysis orientation by default because one test cartridge maps one specimen to up to eight reagents.
- The MVP scope is PCR-specific Excel upload and analysis, not a generic spreadsheet chart builder.
- The default chart visual target is clean, modern, and analysis-readable: close to clinical/report-quality PCR plots, but less dense than the provided reference image.
- The active Threshold-edition identity is IsoAmplar Plot Analysis T for LAMP amplifier result analysis, with developer credit to Jang Si Un. The original non-T app remains separately deployed.
- Native editable Excel charts are excluded for the next refinement; analysis continuity should be handled by an Analysis XLSX project/session file containing the full imported dataset and settings.
- Internal analysis tabs are accepted for working with multiple analyses in one browser window.
- Custom legends should be app-controlled projections rather than ECharts built-in legend state.
- Hover inspection should use a fixed readout outside the plot, with app-controlled curve highlighting by `curveId`; >20 curve assistance should reduce selection or styling without blocking preview/export.
- Export filenames should include the sanitized analysis name when available, while keeping browser-local date and `plotN` / `analysisN` counters.
- Curve labels should use a visible separator (` │ `) rather than `/` so specimen or reagent names containing slash remain readable.
- Dirty close/replace now uses explicit confirmation choices instead of passive warnings or silent overwrite.
- Report-style legend export supports image output and a rich HTML-table clipboard path for Excel cell paste; native editable Excel charts remain excluded.
- Complex plot inspection should use explicit Box zoom that writes Fixed X/Y bounds and supports Previous scale return, not wheel zoom or hidden data cropping.
- The T edition copies the currently filtered Threshold review projection as a four-column rich Excel clipboard table; this remains absent from the original non-T edition.
- Quick Paste Import is implemented for small pasted tables, including a single-specimen mode where the specimen name is supplied before import; it is paste/import only and must not become an in-app spreadsheet editor or source-data correction tool.
- Scale controls persist the editable draft separately from the last valid applied bounds. Invalid active drafts keep the last valid preview and block only plot-bearing image output; valid non-overlap remains warning-only pending UD-15.
- Changed decisions must be superseded, not deleted.
