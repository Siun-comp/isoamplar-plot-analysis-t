# 05 GUI 옵션 제안

## Purpose
PCR 증폭형광 그래프 작성 웹툴의 GUI 방향과 통합 목업 기준을 정리한다.

## Status
Draft for consolidated mockup refinement

## Last Updated
2026-07-07

## Owner
Product / UX / Agent

## Compression-Safe Summary
- 이 문서는 GUI 선택지와 현재 통합 목업 기준을 함께 기록한다.
- 현재 통합 목업은 Option A의 데이터 선택/scale 흐름과 Option D의 데이터 확인 스트립을 합친 구조다.
- 사용자 UI에서는 `열/column/Col`, `반복`, `series`, `solo/isolate` 같은 개발자/분석자 중심 표현을 기본 표시하지 않는다.
- 현재 화면 용어는 `검체 / 시약 / 개별 곡선`, 체크박스 기반 `선택/미선택`, `단독 보기`, `선택만 강조`를 기준으로 둔다.
- 범례 순서 조정, 범례/스타일 프리셋, 그룹 스타일 규칙, 개별 곡선 override를 현재 통합 목업에 반영했다.
- 데이터 선택의 기본 보기는 `시약별`이며, 필요할 때 `검체별` 보기로 전환할 수 있다. 전환해도 선택 상태는 유지된다.
- 설정 패널은 `Scale 설정`, `스타일 설정`, `범례 순서 설정`, `Export 설정` 4개 접힘/펼침 섹션으로 관리한다.
- 사용자가 선택해야 하는 세부 항목은 별도 표시하고 아직 확정하지 않는다.

## Update Rule
GUI 방향, 사용자 흐름, 화면 구성, 또는 전문가 검토 결과가 바뀔 때 갱신한다. 사용자가 최종 선택하기 전에는 `DECISIONS.md`에 UI 결정을 확정 기록하지 않는다.

## 설계 전제
- Excel 업로드를 1차 입력으로 둔다.
- 샘플 구조는 `Sheet1`, 1행 검체명, 2행 시약명, 3행 이후 fluorescence 값이다.
- 별도 X축 정보가 없으면 기본 X축은 `Cycle 1~N`으로 자동 생성한다.
- 각 데이터 묶음은 화면에서 하나의 `개별 곡선`으로 표시한다. 내부 구현명으로 `series`를 사용할 수는 있으나 사용자 UI에는 노출하지 않는다.
- 사용자는 검체, 시약, 개별 곡선 단위로 자유롭게 선택/미선택할 수 있어야 한다.
- 시약은 데이터셋마다 다를 수 있으며 현재 계획상 최대 8종까지를 자연스럽게 다룰 수 있어야 한다.
- 테스트 카트리지 하나는 기본적으로 `1검체 x 최대 8시약` 결과로 이해한다. Excel에서 검체가 먼저 보이는 것은 업로드 구조일 뿐, 기본 분석 탐색 축은 시약이다.
- 검체는 수십~수백 개까지 늘어날 수 있으므로 모든 항목을 한 번에 펼치는 UI를 기본으로 삼지 않는다.
- X/Y scale은 자동, 수동 고정, preset 1, preset 2를 지원할 수 있어야 한다.
- 선택 곡선별 범례명, 색, 선 스타일, 굵기, 표시 여부를 조정할 수 있어야 한다.
- 그래프는 PNG 다운로드를 기본 보장 목표로 하고, 클립보드 복사는 지원 환경에서 제공한다.

## 공통 GUI 원칙
- 첫 화면은 곧바로 작업 화면이어야 하며 랜딩 페이지를 만들지 않는다.
- 업로드 후 앱이 해석한 구조를 사용자가 확인할 수 있어야 한다.
- 검체명 원본 표시와 묶음용 표시명은 분리한다.
- `검체2`와 `검체 2`처럼 유사한 이름은 자동 병합하지 않고 사용자 확인 대상으로 표시한다.
- 필터와 선택/미선택 상태는 항상 눈에 보여야 한다.
- 고정 scale이 켜져 있는 상태는 그래프 주변에서 명확히 보여야 한다.
- 복사 실패 시 PNG 다운로드로 이어지는 fallback을 제공한다.
- 그래프 영역과 설정 패널이 서로 가리지 않아야 한다.
- 원본 증폭형광값에는 smoothing, baseline correction, normalization, 평균화, log scale, Ct/Cq 계산을 자동 적용하지 않는다.
- 변환, 평균선, 대표곡선, Ct/Cq 계산은 사용자가 명시적으로 요청하거나 추후 기능으로 확정할 때만 추가한다.

# 현재 통합 목업 기준

## 통합 방향
사용자 확인사항에 따라 현재 기준은 `Option D 기반 + Option A 작업 방식`이다.

- 업로드 직후 데이터 해석 확인과 경고는 Option D처럼 상단 스트립으로 유지한다.
- 실제 작업은 Option A처럼 왼쪽 데이터 선택, 중앙 그래프, 오른쪽 scale/style/export 패널에서 즉시 조정한다.
- 데이터 선택과 scale 지정은 Option A 방식을 기준으로 한다.
- 사용자 UI에서는 `열`, `column`, `Col A` 같은 원본 열 표현을 쓰지 않는다.
- 사용자에게는 `검체 / 시약 / 개별 곡선`처럼 업무 의미 중심으로 표시한다.
- `반복`은 원본 데이터 또는 사용자의 명시적 정의가 있을 때만 선택 기능으로 검토하며, 현재 임상검체 중심 기본 화면에는 노출하지 않는다.

## 데이터 선택 기준
- 왼쪽 패널은 `시약별` 보기를 기본으로 하고 `검체별` 보기를 보조로 제공한다.
- `검체별` 보기는 `검체 > 시약 > 개별 곡선` 3단 구조다.
- `시약별` 보기는 `시약 > 검체 > 개별 곡선` 3단 구조다.
- 두 보기 전환은 같은 선택 상태를 다른 방식으로 보여주는 것이며, 선택된 곡선을 초기화하지 않는다.
- 선택 상태는 트리 위치가 아니라 안정적인 `곡선 ID` 기준으로 유지한다.
- 검체 체크박스는 하위 시약/곡선을 일괄 제어하고, 시약 체크박스는 해당 시약의 관련 곡선을 일괄 제어한다.
- 부분 선택 상태는 tri-state 개념으로 표시한다.
- `제외/복원` 흐름은 기본 화면에서 쓰지 않고, 체크박스와 `선택/미선택` 상태로 표현한다.
- 수십~수천 개 곡선을 위해 검색, 전체 선택/해제, 선택 반전, 선택만 보기, 미선택 보기, 경고 보기, 대분류 접기/펼치기, scroll 또는 가상 목록을 제공한다.
- 대분류는 양쪽 보기 모두에서 접기/펼치기를 지원한다.
- 검체가 수십~수백 개일 때는 대분류를 보수적으로 접어 두고 검색 결과나 현재 선택이 있는 그룹을 우선 펼친다.
- 검색은 검체명, 시약명, 개별 곡선 표시명을 모두 대상으로 하며, 실제 구현에서는 입력 후 150~250ms 정도의 debounce를 둔다.
- 검색 상태에서 전체 선택/해제 같은 일괄 동작은 현재 화면에 보이는 행만이 아니라 검색 조건에 맞는 전체 곡선에 적용한다.
- 일괄 동작 버튼에는 영향을 받는 곡선 수를 표시하거나 적용 직전 확인을 제공한다.
- 빠른 비교를 위해 `단독 보기`, `선택만 강조`, `선택 항목만 보기`처럼 의미가 직접 드러나는 한국어 액션을 제공한다.

## 설정 패널 기준
- 설정 패널은 `Scale 설정`, `스타일 설정`, `범례 순서 설정`, `Export 설정` 4개 아코디언으로 구성한다.
- 기본 상태는 `Scale 설정`만 펼치고 나머지는 요약 배지로 접어 둔다.
- 사용자가 스타일이나 범례를 조정 중이거나 해당 섹션에 오류가 있으면 해당 섹션을 자동으로 펼칠 수 있다.
- 여러 섹션을 동시에 펼칠 수 있어야 하며, 접힌 섹션에도 현재 핵심 상태를 요약 배지로 표시한다.
- 아코디언 헤더는 실제 구현에서 버튼으로 만들고 `aria-expanded`, `aria-controls`를 제공한다.

## 스타일 적용 기준
기본 UX는 “그룹 규칙 + 개별 override” 구조다.

권장 기본값:

- 검체별 색상
- 시약별 선 종류
- 같은 검체/시약에 여러 곡선이 있을 때는 투명도, 굵기, 또는 마커로 보조 구분
- 개별 곡선 override는 일반 편집에서는 최우선이다.
- 단, 사용자가 프리셋을 적용하면 해당 적용 범위의 기존 개별 override를 덮어쓴다.

스타일 적용 단계:

1. 기본 팔레트와 기본 선 스타일
2. 검체/시약 그룹 규칙
3. 개별 곡선 override
4. 마우스오버 또는 선택 강조 같은 임시 표시

사용자는 아래 조합을 바꿀 수 있어야 한다.

- 검체별 색상
- 시약별 색상
- 검체별 선 종류
- 시약별 선 종류
- 개별 곡선 색상, 선 종류, 굵기, 범례명 override
- 범례/스타일 프리셋을 선택된 데이터에 적용
- 프리셋 적용은 의도적으로 덮어쓰는 동작이므로 적용 전후 영향 범위를 보여주고 되돌리기를 제공하는 것을 권장한다.
- 동일 시약명 매칭으로 검체 1과 검체 2의 같은 시약에 동일 프리셋 적용

## 대량 곡선 가독성 기준
- 그래프에서 모든 곡선을 항상 같은 강도로 보이면 안 된다.
- 마우스오버 또는 선택 강조 상태에서는 선택한 곡선을 강조하고 나머지는 흐리게 표시한다.
- 범례 기본값은 grouped legend로 둔다.
- 범례 모드는 그룹 범례, 전체 곡선 범례, 접기/숨김을 지원한다.
- 전체 곡선 범례를 PNG에 포함하면 그래프 영역이 줄어들 수 있으므로 경고한다.
- 범례 순서는 기본적으로 사용자가 현재 선택한 보기/정렬 순서를 따른다.
- 사용자가 수동으로 범례 순서를 바꾸면 화면과 export가 같은 순서를 따른다.
- 표시 곡선이 20개를 초과하면 분석 가독성이 급격히 떨어지므로 경고와 필터/분할 제안을 표시한다.
- 앱은 많은 곡선을 가져올 수 있지만, 최종 plot은 분석 가능한 표시 곡선 수로 좁히도록 안내한다.

## Scale 기준
- X/Y 각각 자동, 수동 고정, Preset 1, Preset 2를 제공한다.
- 현재 scale mode는 그래프 상단 배지로 표시한다.
- 고정 scale은 필터 변경, 선택 변경, redraw 이후에도 유지한다.
- min/max 유효성 검사, 자동 재설정, 현재 데이터 범위 가져오기를 제공한다.
- preset 적용 시 범위 밖 점 수와 영향을 받은 곡선 수를 표시한다.

# Option A. 한 화면 작업대형

## 개요
업로드, 데이터 선택, 그래프 미리보기, 스타일 설정, export를 한 화면에서 모두 처리하는 전문가 작업대형 UI다.

## 추천 대상
- PCR 그래프를 자주 조정하는 사용자
- 검체/시약/개별 곡선을 자주 켜고 끄는 사용자
- scale과 스타일을 보면서 빠르게 결과물을 맞추려는 사용자

## 화면 구성

```text
┌─────────────────────────────────────────────────────────────────────────────┐
│ 상단 툴바: 파일 업로드 | 추가 데이터 | 시트 | 그래프 제목 | 복사 | PNG 저장 │
├───────────────┬───────────────────────────────────────┬─────────────────────┤
│ 데이터 패널    │ 그래프 미리보기                         │ 설정 패널            │
│               │                                       │                     │
│ 검색/요약      │ [구조: 1행 검체, 2행 시약, Cycle 1~45]   │ 축 설정              │
│ 선택 4 미선택 0│ [Scale: Y Preset 1] [범위 밖 점 0개]     │ X: 자동/고정/P1/P2   │
│               │                                       │ Y: 자동/고정/P1/P2   │
│ 검체 필터      │                                       │                     │
│ □ 검체 1       │        PCR Amplification Plot          │                     │
│ □ 검체2        │                                       │                     │
│ □ 검체 2       │                                       │                     │
│               │                                       │ 범례/스타일          │
│ 시약 필터      │                                       │ 색상, 선, 굵기       │
│ □ A1 □ A2      │                                       │                     │
│               │                                       │ 레이아웃             │
│ 곡선 목록      │                                       │ 크기, 범례 위치      │
│ ☑ 검체1/A1     │                                       │                     │
│ ☑ 검체1/A2     │                                       │                     │
├───────────────┴───────────────────────────────────────┴─────────────────────┤
│ 하단 데이터 해석 바: 1행 검체 / 2행 시약 / 3행부터 데이터 / Cycle 1~45       │
└─────────────────────────────────────────────────────────────────────────────┘
```

## 주요 동작
- Excel 업로드 즉시 PCR 구조를 감지하고 그래프를 렌더링한다.
- 왼쪽 패널에서 검체/시약/개별 곡선을 선택하거나 미선택한다.
- 중앙 그래프는 변경 즉시 반영한다.
- 오른쪽 패널에서 scale preset, 범례, 색, 선 스타일을 수정한다.
- 상단에서 클립보드 복사와 PNG 저장을 수행한다.

## 장점
- 가장 빠른 조정 흐름.
- 그래프를 보면서 필터, scale, 스타일을 동시에 조정 가능.
- 고급 사용자가 원하는 자유도가 높다.

## 단점
- 첫 사용자에게는 화면이 복잡하게 보일 수 있다.
- 작은 화면에서는 3분할 구조가 답답할 수 있다.
- 기능이 늘면 패널 정리가 중요하다.

## 품질 기준
- 그래프 영역은 최소 60% 이상의 가로 공간을 확보한다.
- 왼쪽 selection 패널은 검색과 전체 선택/해제를 제공한다.
- 왼쪽 selection 패널은 `검체 > 시약 > 개별 곡선` 3단 구조를 지원한다.
- 선택된 곡선 수, 미선택 곡선 수, 현재 필터 조건을 요약 배지로 표시한다.
- 미선택 항목만 보기 기능을 제공한다.
- 오른쪽 설정 패널은 섹션 접기/펼치기를 제공한다.
- 현재 적용 중인 scale mode를 그래프 상단 배지로 표시한다.
- preset 적용 시 보이지 않는 데이터가 생기면 `N개 점이 현재 축 범위 밖`처럼 경고한다.

# Option B. 단계형 마법사 + 최종 편집형

## 개요
사용자가 순서대로 업로드, 구조 확인, 데이터 선택, 그래프 설정, export를 진행하는 안내형 UI다.

## 추천 대상
- PCR 그래프 작성 흐름을 처음 사용하는 사용자
- 잘못된 Excel 구조나 선택 실수를 줄이고 싶은 사용자
- 단일 그래프를 안정적으로 출력하는 것이 중요한 사용자

## 화면 구성

```text
┌──────────────────────────────────────────────────────────────────────┐
│ 단계 표시: 1 업로드 > 2 데이터 확인 > 3 선택 > 4 스타일 > 5 출력      │
├──────────────────────────────────────────────────────────────────────┤
│ 현재 단계 콘텐츠                                                     │
│                                                                      │
│ [1 업로드] Excel 드롭존, 샘플 구조 안내                               │
│ [2 확인] 검체 행/시약 행/데이터 시작 행 확인                          │
│ [3 선택] 검체, 시약, 개별 곡선 선택                                  │
│ [4 스타일] scale, 범례, 색, 선 스타일                                 │
│ [5 출력] 그래프 미리보기, 클립보드 복사, PNG 저장                     │
├──────────────────────────────────────────────────────────────────────┤
│ 하단: 이전 | 다음 | 현재 단계 검증 메시지                             │
└──────────────────────────────────────────────────────────────────────┘
```

## 주요 동작
- 각 단계에서 필요한 설정만 보여준다.
- 데이터 구조가 모호하면 2단계에서 멈추고 사용자가 확인한다.
- 3단계에서 선택된 개별 곡선만 4단계 스타일 편집 대상으로 보여준다.
- 5단계에서 최종 그래프와 출력 버튼을 제공한다.
- 최종 화면에서 “편집으로 돌아가기”를 제공한다.

## 장점
- 실수 방지에 강하다.
- Excel 입력 구조가 복잡해져도 안내하기 쉽다.
- 신규 사용자에게 가장 이해하기 쉽다.

## 단점
- 자주 조정하는 작업 속도는 Option A보다 느리다.
- 그래프를 보며 즉시 필터를 바꾸는 흐름이 덜 자연스럽다.
- 고급 사용자는 단계 이동을 답답하게 느낄 수 있다.

## 품질 기준
- 각 단계는 하나의 핵심 판단만 요구한다.
- 2단계 데이터 해석 화면에서 원본 미리보기를 반드시 제공한다.
- 2단계에서 `3~47행 -> Cycle 1~45`처럼 데이터 행과 cycle 생성 규칙을 표시한다.
- 빈 행, 중간 누락, 숫자가 아닌 값, 서로 다른 곡선 길이는 자동 보정하지 않고 경고한다.
- 5단계 최종 화면에서는 그래프와 export 버튼을 한 화면에 둔다.
- 사용자가 언제든 이전 단계로 돌아가도 설정이 유지되어야 한다.

# Option C. 비교/배치 보드형

## 개요
여러 그래프 또는 여러 scale preset을 비교하며 배치하는 보드형 UI다. “배치” 기능을 강하게 반영하는 방향이다.

## 추천 대상
- 여러 검체/시약 조합을 한 번에 비교하는 사용자
- 같은 데이터에서 scale preset 1과 preset 2를 비교해야 하는 사용자
- 보고서용 다중 그래프 이미지를 구성하려는 사용자

## 화면 구성

```text
┌────────────────────────────────────────────────────────────────────────────┐
│ 상단 툴바: 업로드 | 추가 데이터 | 새 그래프 | 레이아웃 | 복사 | PNG 저장     │
├──────────────────────┬─────────────────────────────────────────────────────┤
│ 데이터/선택 패널       │ 그래프 보드                                           │
│                      │ ┌────────────────────┬────────────────────┐          │
│ 검체 그룹             │ │ Graph 1             │ Graph 2             │          │
│ 시약 그룹             │ │ 검체1 A1/A2          │ 검체2 A1/A2          │          │
│ 곡선 목록             │ │ Scale preset 1      │ Scale preset 2      │          │
│                      │ └────────────────────┴────────────────────┘          │
│ Scale presets         │ ┌─────────────────────────────────────────┐          │
│ P1: X/Y min/max       │ │ Selected graph detail                    │          │
│ P2: X/Y min/max       │ └─────────────────────────────────────────┘          │
├──────────────────────┴─────────────────────────────────────────────────────┤
│ 하단 선택 그래프 설정: 범례, 색상, 선 스타일, 제목, 축 라벨                  │
└────────────────────────────────────────────────────────────────────────────┘
```

## 주요 동작
- 하나의 데이터셋에서 여러 그래프 카드를 만든다.
- 각 그래프는 선택된 검체/시약/개별 곡선과 scale preset을 가질 수 있다.
- 보드에서 그래프를 선택하면 하단 설정 패널이 해당 그래프 기준으로 바뀐다.
- 보드 전체 PNG 또는 선택 그래프 PNG를 저장할 수 있다.
- 클립보드는 우선 선택 그래프 복사를 기본으로 한다.

## 장점
- 비교와 배치 목적에 가장 강하다.
- scale preset 2개를 실제 비교 화면으로 보여주기 좋다.
- 보고서용 결과물 제작에 적합하다.

## 단점
- MVP 구현 난이도가 가장 높다.
- 사용자에게 “그래프 카드”와 “개별 곡선” 개념을 설명해야 한다.
- 클립보드 복사 범위가 선택 그래프인지 보드 전체인지 결정이 필요하다.

## 품질 기준
- 각 그래프 카드의 제목, scale mode, 선택 곡선 수를 표시한다.
- 선택 그래프와 전체 보드 export를 명확히 분리한다.
- 그래프 카드 크기 변경 시 축 라벨과 범례가 잘리지 않아야 한다.
- 보드 기능이 MVP에 포함될 경우 E2E와 시각 회귀 테스트가 필요하다.
- 전체 보드 PNG는 선택 그래프 PNG보다 품질 리스크가 크므로 별도 검증 기준을 둔다.

# Option D. 추천 혼합형

## 개요
전문가 검토에서 가장 균형적이라고 판단한 추천안이다. Option A의 빠른 한 화면 작업 흐름을 기본으로 하고, Option B의 업로드 직후 데이터 해석 확인 단계를 상단 또는 모달 형태로 흡수한다.

## 추천 대상
- 대부분의 초기 MVP 사용자
- Excel 업로드 구조 오류를 줄이면서 빠른 조정을 원하는 사용자
- 단일 그래프 중심으로 시작하되 추후 비교/배치 기능 확장을 고려하는 사용자

## 화면 구성

```text
┌─────────────────────────────────────────────────────────────────────────────┐
│ 상단 툴바: 업로드 | 추가 데이터 | 데이터 확인 | 복사 | PNG 저장             │
├─────────────────────────────────────────────────────────────────────────────┤
│ 데이터 해석 확인 스트립: Sheet1 / 1행 검체 / 2행 시약 / 3~47행 -> Cycle 1~45 │
│ 경고: 유사 검체명 1건 확인 필요 | Scale: Y Preset 1 | 선택 4 미선택 0        │
├────────────────────┬─────────────────────────────────────┬──────────────────┤
│ 선택 패널           │ 그래프 미리보기                       │ 설정 패널          │
│ 검색                │ PCR Amplification Plot                │ 축 preset          │
│ 검체 > 시약 > 곡선   │                                     │ 범례/스타일        │
│ 미선택 항목 보기     │                                     │ export 품질        │
└────────────────────┴─────────────────────────────────────┴──────────────────┘
```

## 주요 동작
- 첫 업로드 직후 데이터 해석 확인 스트립 또는 모달을 보여준다.
- 사용자가 구조 확인을 완료하면 한 화면 작업대형으로 전환된다.
- 데이터 해석 확인은 언제든 상단 `데이터 확인` 버튼으로 다시 열 수 있다.
- 검체/시약/개별 곡선 선택, scale, 스타일, export는 한 화면에서 즉시 조정한다.
- 추후 Option C형 보드 기능을 붙일 수 있도록 현재 그래프 설정을 독립된 chart configuration으로 관리한다.

## 장점
- Option A의 빠른 조정 속도를 유지한다.
- Option B의 입력 오류 방지 장점을 흡수한다.
- MVP 구현 리스크와 사용자 학습 비용의 균형이 좋다.
- 추후 비교/배치 기능으로 확장하기 쉽다.

## 단점
- Option A보다 상단 상태 영역 설계가 조금 더 복잡하다.
- 데이터 확인 스트립/모달의 노출 타이밍을 잘못 잡으면 사용자가 번거롭게 느낄 수 있다.
- 비교/배치 기능은 아직 직접 제공하지 않는다.

## 품질 기준
- 데이터 해석 확인은 업로드 직후 1회 강하게 노출하고 이후에는 접을 수 있어야 한다.
- 유사 검체명, cycle 길이 불일치, 숫자가 아닌 값은 상단 경고로 계속 접근 가능해야 한다.
- scale preset과 범위 밖 데이터 경고는 그래프 가까이에 유지한다.
- export 품질 설정은 기본값을 제공하되 고급 설정으로 접어 둔다.
- 이 안을 선택하더라도 “배치”의 의미는 별도 사용자 결정으로 남긴다.

# 공통 보강사항

전문가 검토 후 모든 안에 적용해야 하는 명백한 보강사항이다.

- 데이터 해석 확인 영역을 둔다.
- 이름이 유사한 검체를 자동 병합하지 않고 사용자 확인 대상으로 둔다.
- X축이 없을 때 `Cycle 1~N` 자동 생성 상태를 표시한다.
- 데이터 행과 cycle의 연결을 명확히 표시한다. 예: `3~47행 -> Cycle 1~45`.
- 빈 행, 중간 누락, 숫자가 아닌 값, 서로 다른 곡선 길이는 자동 보정하지 않고 경고한다.
- 검체/시약 grouping은 `검체 > 시약 > 개별 곡선`과 `시약 > 검체 > 개별 곡선` 두 가지 보기로 제공한다.
- 두 보기의 선택 상태는 동일한 `곡선 ID` 집합을 공유하므로 보기 전환이 선택 초기화를 일으키면 안 된다.
- 검체 수십~수백 개, 시약 최대 8종, 곡선 수천 개까지 대비해 실제 구현은 평면화된 visible tree와 가상 목록을 사용한다.
- 대분류 접기/펼치기, 전체 대분류 접기/펼치기, 검색 결과가 있는 대분류 자동 펼치기를 제공한다.
- 동일 검체/시약 조합이 여러 개면 `개별 곡선`으로 표시하고, 반복/복제 측정 여부는 사용자 확인 전에는 단정하지 않는다.
- 평균선이나 대표곡선은 자동 생성하지 않는다.
- 선택/미선택 상태를 숨기지 않는다.
- 선택된 곡선 수, 미선택 곡선 수, 현재 필터 조건을 요약 표시한다.
- selection 패널에는 검색, 전체 선택/해제, 미선택 항목만 보기를 제공한다.
- 고정 scale 적용 상태를 그래프 가까이에 표시한다.
- preset scale 적용으로 그래프 밖으로 벗어나는 점이 있으면 경고한다.
- 곡선 스타일은 개별 설정뿐 아니라 검체별 색상, 시약별 선 종류, 동일 조합 곡선의 투명도/굵기 같은 그룹 단위 적용을 고려한다.
- 곡선이 많을 때를 대비해 grouped legend, 범례 검색/접기, export 시 범례 포함 여부를 제어할 수 있어야 한다.
- PNG 저장을 항상 제공하고, 클립보드 복사는 실패 fallback을 둔다.
- PNG export 품질 기준을 둔다: 기본 흰 배경, 제목/축/범례 포함, 화면 설정과 동일한 결과, 최소 2x 배율 후보, 명확한 파일명 규칙.
- chart export는 일반 화면 캡처보다 chart library의 공식 export API 또는 chart config 기반 재렌더를 우선한다.
- 추가 데이터 입력 시 기존 곡선과 새 곡선의 출처 파일을 구분한다.
- 추가 데이터 입력 시 cycle 수 불일치, 검체명 충돌, 시약명 충돌을 검사한다.
- 모든 checkbox/input은 명확한 label을 가져야 하며, 키보드 조작과 focus 이동이 가능해야 한다.
- 트리 UI는 방향키 이동, 오른쪽 방향키 펼치기, 왼쪽 방향키 접기, Space 선택 토글을 고려한다.
- 아코디언 섹션은 버튼 헤더와 `aria-expanded`, `aria-controls`를 제공한다.
- 색상만으로 곡선을 구분하지 않고 선 스타일, 범례명, 마우스오버/선택 강조 상태를 함께 사용한다.
- 데스크톱 전체 편집을 기본으로 하되, 태블릿 이하에서는 패널을 drawer 또는 tabs로 접는 설계를 고려한다.

## 전문가 검토 반영 요약
- UX/Product 관점: 자주 조정하는 흐름 때문에 Option A가 가장 적합하며, Option B의 업로드/해석 확인을 Option A에 흡수하는 혼합형이 좋다.
- PCR/qPCR 시각화 관점: Cycle 생성 규칙, 개별 곡선 표시, 그룹 단위 스타일, scale 범위 밖 데이터 경고, 데이터 왜곡 방지 원칙이 필요하다.
- Frontend/QA 관점: export 품질 기준, 접근성, 반응형 기준, clipboard fallback, Option C 시각 회귀 테스트 필요성을 명시해야 한다.
- 대량 데이터 관점: 검체가 수백 개일 수 있으므로 검체별/시약별 보기 전환, 대분류 접힘, 검색, 가상 목록, 선택 상태와 트리 표시의 분리가 필요하다.

## 사용자 결정 필요
아래 항목은 문서에 반영만 하고 아직 결정하지 않는다.

- 현재 통합 목업을 최종 구현안으로 확정할지, 추가 사용성 수정 후 확정할지
- 대량 데이터에서 기본 펼침 기준을 첫 그룹만 펼침, 선택된 그룹만 펼침, 검색 결과 그룹만 펼침 중 무엇으로 둘지
- `검체2`와 `검체 2`를 같은 검체로 묶을지 여부
- “배치”의 의미: 다중 그래프 보드, 일괄 출력, 또는 단순 레이아웃 조정
- 클립보드 복사 기본 범위: 현재 미리보기, 선택 그래프, 또는 향후 전체 보드
- MVP에서 추가 데이터 입력을 Excel 추가 업로드까지만 할지, 붙여넣기/직접 입력까지 포함할지
- Cycle 시작값을 항상 1로 할지, 사용자가 Cycle 0 시작을 선택할 수 있게 할지
- 동일 검체/시약 조합이 여러 개일 때 이를 단순 개별 곡선으로만 볼지, 사용자가 별도 묶음 의미를 부여할 수 있게 할지
- scale preset 1/2의 목적을 표준축, 확대축, 비교축 중 무엇으로 둘지
- 그룹 규칙을 바꿀 때 기존 개별 override를 유지할지 초기화할지
- 범례 기본 모드를 grouped legend로 둘지, 전체 곡선 범례로 둘지
- 수십 개 곡선을 export할 때 전체 범례를 포함할지 그룹 범례만 포함할지
- Y축 음수 허용 여부
- PNG export 해상도, 투명 배경, 파일명 규칙을 사용자가 조정할 수 있게 할지
- SVG 저장을 MVP에 포함할지 여부
- 모바일 화면 최적화를 MVP에 포함할지 여부

## 1차 추천
현재 요구사항과 구현 안정성을 기준으로는 Option D를 1차 MVP 후보로 추천한다.

이유:

- PCR 그래프 조정은 업로드 후 선택/미선택/scale/style을 자주 바꾸는 흐름이 많다.
- 사용자가 원하는 자유로운 선택과 미선택에 가장 잘 맞는다.
- Option A의 빠른 조정 속도와 Option B의 입력 검증 안정성을 함께 가져간다.
- Option C보다 구현 위험이 낮고, 추후 비교/배치 기능으로 확장 가능하다.

Option D는 `Option A + Option B의 업로드/데이터 해석 확인 일부 흡수`를 실제 선택지로 정리한 추천안이다. 단, 이는 최종 결정이 아니라 추천이다.

사용자가 순수 조정 속도를 최우선으로 보면 Option A, 실수 방지와 안내를 더 중요하게 보면 Option B, 여러 그래프 배치/비교가 핵심이면 Option C가 적합하다.

## 전문가 검토 로그
- UX/Product 검토 완료: Option A 기반, Option B식 첫 업로드 검증 흡수를 추천.
- PCR/qPCR 시각화 검토 완료: Cycle 생성 규칙, 개별 곡선 표시, 그룹 스타일, scale 범위 밖 경고, 데이터 왜곡 방지 원칙을 보강해야 한다고 판단.
- Frontend/QA 검토 완료: export 품질 기준, 접근성, 반응형, clipboard fallback, Option C 시각 회귀 테스트 리스크를 보강해야 한다고 판단.
- 2차 UX/Product 검토 완료: A/D 통합, 원본 위치 표현 제거, 그룹 규칙 + 개별 override, 대량 곡선 검색/필터/접기/단독 보기를 반영해야 한다고 판단.
- 2차 PCR/qPCR 검토 완료: `검체=색상`, `시약=선 종류`, 개별 곡선 override 최우선, grouped legend, scale 범위 밖 영향 표시를 반영해야 한다고 판단.
- 3차 UX/Product 및 PCR/qPCR 검토 반영: `반복`, `series`, `solo/isolate`, `제외/복원`은 기본 사용자 UI에서 제거하고, 범례 순서 조정과 범례/스타일 프리셋을 현재 목업에 추가했다.
- 4차 UX/Product 검토 반영: `검체별/시약별` 보기 전환, 대분류 접기/펼치기, 시약 최대 8종과 검체 수백 개를 고려한 보수적 기본 펼침이 필요하다고 판단.
- 4차 Frontend/QA 검토 반영: 선택 상태는 곡선 ID 기준으로 유지하고, `검체 -> 시약 -> 곡선` 및 `시약 -> 검체 -> 곡선` 파생 인덱스를 만들어 같은 데이터를 두 가지 트리로 표시해야 한다고 판단.
- 5차 사용자 결정 및 전문가 검토 반영: 기본 보기는 `시약별`, 검색 일괄 동작은 검색 조건 전체 대상, 프리셋 적용은 기존 개별 설정 덮어쓰기, 범례/export 순서는 현재 사용자 순서 기준, 20개 초과 표시 곡선은 가독성 경고 대상으로 확정했다.
